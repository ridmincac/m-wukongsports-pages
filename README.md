# m-wukongsports-pages

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内容不针对任何特定域名或具体网站，主要用于存放一些静态页面资源，方便历史版本管理和快速访问。

## 项目简介

本仓库旨在提供一种轻量的方式，将独立编写的 HTML 页面集中存储，并利用 GitHub Pages 等静态托管服务进行发布。页面内容涵盖不同类型的展示页、工具页或信息页，适合需要简单部署和长期维护的场景。

## 目录说明

```
.
├── index.html          # 仓库首页（可选）
├── pages/              # 存放所有独立 HTML 页面的目录
│   ├── example.html
│   └── ...
└── assets/             # 公共资源文件（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 文件代表一个独立的页面，文件命名应简洁、具有描述性。
- `assets/`：存放页面共用的静态资源，按类型划分子目录。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖外部数据库或后端服务。
- 页面之间无强关联，可单独访问。
- 归档时请保持文件结构清晰，避免路径混乱。
- 如需引用公共资源，请使用相对路径，例如 `../assets/css/style.css`。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保资源引用正确。
- 更新页面：直接修改对应文件，注意保留历史版本（可通过 Git 提交记录回溯）。
- 删除页面：移除多余文件，并检查是否有内部链接需要清理。
- 不建议在此仓库中存放动态内容或需要后端支持的功能。

## 使用方式

1. 克隆仓库到本地。
2. 将需要归档的 HTML 页面放入 `pages/` 目录。
3. 根据需要调整资源引用路径。
4. 提交并推送至 GitHub，启用 Pages 服务后即可通过仓库 Pages 地址访问。
